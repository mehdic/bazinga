### r/programming

**Title:** [Open Source] Multi-agent orchestration system for parallel AI development

**Body:**
```
I built a system that coordinates teams of AI coding agents to work in parallel, similar to how human dev teams split up work.

## Motivation

AI coding assistants (Claude Code, Cursor, etc.) are powerful but work sequentially. When you need multiple features:
- Traditional: Build feature 1 → then 2 → then 3 (60 minutes)
- With orchestration: Build all 3 in parallel (20 minutes)

The challenge is coordination—ensuring tasks are independent, managing dependencies, routing messages between agents.

## Architecture

5 specialized agents:
1. **Project Manager** - Analyzes requirements, decides parallelism (1-4 devs based on independence)
2. **Developers** - Implement code, write tests, run quality checks (parallel execution)
3. **QA Expert** - Integration and E2E testing (optional advanced mode)
4. **Tech Lead** - Code review, security validation, architecture checks
5. **Orchestrator** - Message routing, workflow state management

## Technical Highlights

- **Adaptive parallelism algorithm** - Analyzes task independence, file overlap, dependencies
- **Automatic quality gates** - Security (bandit/gosec/npm-audit), linting (ruff/eslint), coverage (pytest-cov/jest)
- **Model escalation** - Sonnet for fast iteration, auto-upgrade to Opus at revision 3
- **State management** - JSON-based coordination files (gitignored except config)
- **Multi-language** - Python, JavaScript/TypeScript, Go, Java, Ruby

## Example Usage

```bash
# Install
uvx --from git+https://github.com/mehdic/bazinga.git bazinga init my-project

# Use
@orchestrator implement REST API with auth, user management, and admin endpoints
```

Output:
```

Developer 3 (Admin): ✓ Implementation ✓ Tests ✓ Security ✓ Lint ✓ Coverage

Tech Lead: Reviewing all implementations...
PM: BAZINGA! All features complete in 20 minutes
```

## Performance

Tested on various project types:
- 2 independent features: 2.7x faster
- 3 independent features: 3x faster
- 4 independent modules: 3x faster

Limitations: Max 4 devs (coordination overhead), requires task independence, sequential if dependencies exist.

Early stage—looking for feedback and contributors.

# I Built Parallel AI Developers So You Don't Have to Babysit Agents

If you've used AI coding assistants like Claude Code or Cursor, you've probably experienced this:

You: "Implement feature A"
AI: *works for 20 minutes*
You: "Now implement feature B"
AI: *works for 20 minutes*
You: "Wait, did I run tests? Let me check..."
You: "Okay, feature C now"
AI: *works for 20 minutes*

**Total time: 60+ minutes. Your role: Project manager.**

I got tired of this, so I built **BAZINGA**—a system that coordinates teams of AI agents to work in parallel.

## The Problem: You're the Project Manager

When you use AI coding tools, you end up doing a lot of coordination work:
- Breaking down tasks manually
- Waiting for sequential completion
- Remembering to run tests and security scans
- Checking if one agent is done before starting another
- Managing context and state between tasks

**You wanted to code, but you became a PM.**

## The Solution: Let AI Be the PM

BAZINGA introduces a Project Manager agent that handles all coordination:

```bash
@orchestrator implement JWT auth, user registration, and password reset
```

Behind the scenes:
1. PM analyzes the request
2. Detects 3 independent tasks
3. Spawns 3 developer agents in parallel
4. Routes messages between them
5. Each dev implements + writes tests + runs security scans
6. Tech Lead reviews all work
7. PM confirms completion

**Result: 18 minutes instead of 60.**

## How It Works

BAZINGA coordinates 5 specialized agents:

### 1. Project Manager
- Analyzes your requirements
- Breaks down work into tasks
- Decides parallelism (1-4 devs based on independence)
- Tracks progress and confirms completion

### 2. Developers (1-4 in parallel)
- Implement features
- Write unit tests
- Run security scans
- Check code quality
- Fix issues from reviews

### 3. QA Expert (optional)
- Integration tests
- Contract tests
- E2E scenarios
- Pattern analysis

### 4. Tech Lead
- Code review
- Security validation
- Architecture checks
- Provides feedback or approval

### 5. Orchestrator
- Routes messages between agents
- Maintains workflow state
- Prevents role drift

## Adaptive Parallelism

The PM isn't blindly parallel—it's smart about when to parallelize:

| Request | Decision | Reasoning |
|---------|----------|-----------|
| "Implement JWT auth" | 1 developer | Single feature |
| "Add auth, user mgmt, admin dashboard" | 3 developers | Independent features |
| "Add login and password reset" | 1 developer | Password reset depends on auth |
| "Refactor database layer" | 1 developer | High file overlap |

It analyzes:
- Task independence
- File overlap
- Dependencies
- Complexity vs coordination overhead

## Automatic Quality Gates

Every feature automatically gets:

**Security Scanning:**
- Tools: bandit (Python), gosec (Go), npm audit (JS), brakeman (Ruby)
- Checks: SQL injection, XSS, hardcoded secrets, auth bypasses
- Escalation: Basic scan → Advanced scan after 2 revisions

**Lint Checking:**
- Tools: ruff (Python), eslint (JS), golangci-lint (Go), rubocop (Ruby)
- Checks: Code style, complexity, best practices
- Enforcement: All issues must be fixed before approval

**Test Coverage:**
- Tools: pytest-cov (Python), jest (JS), go test (Go), simplecov (Ruby)
- Checks: Line coverage, branch coverage, untested paths
- Target: 80% coverage by default

## Intelligent Model Escalation

Stuck on a hard problem? BAZINGA automatically escalates:

- **Revision 1-2:** Claude Sonnet (fast, handles 90% of work)
- **Revision 3+:** Claude Opus (deep analysis for persistent issues)

Cost-effective and automatic—no manual switching needed.
