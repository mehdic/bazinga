1. What is BAZINGA?

BAZINGA transforms Claude Code into a multi-agent development team. Instead of one AI doing everything, specialized agents collaborate:

PM → Developer → QA → Tech Lead → PM
  • Project Manager - Analyzes requirements, creates task groups, signals completion
  • Developer - Implements code and writes tests
  • QA Expert - Runs integration and E2E tests
  • Tech Lead - Reviews code quality and security

When the PM is satisfied that all requirements are met, it sends "BAZINGA!" - your signal that the work is complete.

2. Prerequisites

Requirements
  • Claude Code installed and working
  • A project directory with some code
  • BAZINGA files in your project (via bazinga install or manual setup)

Verify Your Setup

Check that the BAZINGA slash commands are available:

Claude Code
/bazinga.orchestrate

If this command is recognized, you're ready to go. If not, ensure the .claude/commands/bazinga.orchestrate.md file exists in your project.

3. Run Your First Orchestration

Let's run a simple orchestration. In Claude Code, type:

Claude Code
/bazinga.orchestrate Add a health check endpoint at /api/health that returns { "status": "ok", "timestamp": "ISO date" }
Good First Tasks

Start with something concrete and self-contained:

  • Add a utility function
  • Create an API endpoint
  • Add input validation
  • Write a new test file

What Happens Next

Once you hit enter, BAZINGA springs into action:

Orchestrator Initializes

The orchestrator loads configuration and spawns the Project Manager.

PM Analyzes Requirements

The PM breaks down your request into task groups and decides execution mode (simple or parallel).

Developer Implements

A Developer agent writes the code and tests.

QA Tests (if applicable)

If integration tests exist and testing mode is "full", QA Expert runs them.

Tech Lead Reviews

Tech Lead runs security scans and reviews code quality.

PM Validates & Sends BAZINGA

PM verifies success criteria are met and signals completion.

4. Understand the Output

During orchestration, you'll see status messages like this:

Orchestration Transcript
ORCHESTRATOR: Initializing session bazinga_20251205_143022...
ORCHESTRATOR: Phase 1 - Spawning Project Manager...
PM: Analyzing requirements... Created 1 task group. Mode: SIMPLE
ORCHESTRATOR: Phase 2 - Spawning Developer for group US1...
DEVELOPER: Implementing health check endpoint... Status: READY_FOR_REVIEW
ORCHESTRATOR: Routing to Tech Lead...
TECH LEAD: Running security scan... Code review passed. Status: APPROVED
PM: All success criteria verified. BAZINGA!

Key Status Codes

Status Meaning Next Step
READY_FOR_QA Developer finished with tests Routes to QA Expert
READY_FOR_REVIEW Developer finished without tests Routes to Tech Lead
PASS QA tests passed Routes to Tech Lead
APPROVED Tech Lead approved code Routes back to PM
BAZINGA All done! Project complete

Hard Rules to Remember

  • Don't interrupt - Let BAZINGA finish; agents handle retries automatically
  • BAZINGA = done - When PM sends BAZINGA, the work is complete
  • Failures are normal - Seeing retries and escalations is expected behavior
