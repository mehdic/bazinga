Meet Your AI Dev Team

BAZINGA coordinates specialized agents, each with distinct responsibilities.

PM → Developer → QA Expert → Tech Lead → PM → BAZINGA!
Agent Role Model
PM Plans tasks, decides execution mode, sends BAZINGA on completion Opus
Developer Implements code and writes tests Haiku
SSE Handles escalated complexity and security-sensitive code Sonnet
QA Expert Runs integration, contract, and E2E tests with 5-level challenges Sonnet
Tech Lead Reviews code, runs security scans, approves or requests changes Opus
Investigator Deep root cause analysis for complex blockers Opus

Hard Rules to Remember

  • MAX 4 parallel developers - Groups beyond 4 are split into phases
  • QA Expert only runs when tests exist AND testing mode = full
  • Orchestrator coordinates only - It never implements code directly
  • PM alone sends BAZINGA - Only after validator acceptance
  • Security tasks force Senior Software Engineer + mandatory Tech Lead review
